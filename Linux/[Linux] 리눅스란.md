# 리눅스란
- 리눅스란 무엇인가?

- 개발을 하다보면 리눅스 서버라는 것을 자연스럽게 접하게 될 것이다.

- 말 그대로 운영체제가 리눅스인 컴퓨터를 말하는 것인데, 자연스럽게 접하다 보니 정확하게 리눅스란 것이 무엇인지<br>
생각해본적이 단 한 번도 없는 것 같아 정리를 해보려 한다.

- 리눅스는 무료 오픈소스 운영체제(OS)의 커널이다.
<br>

### 운영체제와 커널
- 운영체제: 하드웨어와 소프트웨어를 관리하는 프로그램

- 커널: OS의 핵심으로, 하드웨어와 직접 통신하며 메모리, 프로세스, 디바이스를 관리

- 리눅스는 커널이고, 리눅스 배포판(우분투, CentOS 등)은 리눅스 커널 + 유틸리티/도구를 모아 만든 OS
<br>
<hr>
<br>

## ✔️ 리눅스의 탄생 배경
- 1991년 핀란드의 리누스 토발즈가 만듬

- 기존 유닉스와 달리 무료로 사용 가능

- 오픈소스라 누구나 코드를 보고 수정 가능

- 전 세계 개발자들이 함께 발전시킴
<br>
<hr>
<br>

## ✔️ 리눅스의 특징
- 오픈소스

- 멀티태스킹

- 멀티유저

- 안정성과 보안: 서버 환경에서 많이 사용

- 클라이언트/서버 모델: 서버용 OS로 널리 사용
<br>
<hr>
<br>

## ✔️ 개발자가 리눅스를 알아야 하는 이유
### 서버 환경
- 웹 서버 대부분이 리눅스 기반

- 클라우드(AWS, GCP, Azure 등) 서버도 리눅스가 많음
<br>

### 도커/컨테이너
- 대부분 리눅스 컨테이너로 동작

- `Dockerfile` 작성과 컨테이너 관리에 필요
<br>

### DevOps 도구
- CI/CD, Kubernetes 등이 리눅스 환경에서 주로 동작
<br>

### 개발 도구
- Git, Node.js, Python 등이 리눅스에서 유연하게 동작
<br>
<hr>
<br>

## ✔️ 리눅스 배포판
- 우분투: 초보자에게 친화적, 문서/커뮤니티 풍부

- CentOS/RHEL: 기업 서버 환경에서 널리 사용

- 데비안: 안정성 중심

- Amazon Linux: AWS 전용
<br>
<hr>
<br>

## ✔️ 리눅스의 핵심 개념들
### 터미널(Terminal)
- 텍스트 기반 명령 입력 인터페이스

- 개발자는 터미널로 작업 수행
<br>

### 파일 시스템
- 계층적 디렉토리 구조(`/`, `/home`, `/etc` 등)

- 모든 것이 파일로 취급됨
<br>

### 권한(Permission)
- 파일/디렉토리 읽기(r), 쓰기(w), 실행(x)

- 소유자/그룹/기타 사용자별 권한 관리
<br>

### 프로세스
- 실행중인 프로그램

- `ps`, `top` 등으로 확인 가능
<br>
<hr>
<br>

## ✔️ 정리
- 리눅스는 서버와 개발 환경에서 널리 쓰이는 오픈소스 운영체제의 커널

- 개발자는 배포, 인프라, DevOps 도구와의 연계에서 리눅스 이해가 중요
